# 附錄 X：VulnBot 復現步驟與驗證指標

## A. 實驗環境

1. 作業系統：Ubuntu Linux（建議 22.04+）
2. Python：`3.11.x`（本實驗為 `3.11.14`）
3. 套件管理：`pip`
4. 必要外部服務：
- MySQL（需可連線）
- （可選）Milvus（僅在啟用 RAG 時需要）
- 可用 LLM API 端點（OpenAI 相容接口）
- Kali 測試主機（供 SSH 連線）

## B. 原始碼與安裝

1. 取得專案

```bash
git clone git@github.com:youngleetw/VulnBot.git
cd VulnBot
```

2. 建立虛擬環境並安裝相依

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

3. 設定檔確認（四個 YAML）
- `basic_config.yaml`
- `db_config.yaml`
- `kb_config.yaml`
- `model_config.yaml`

## C. 設定重點

1. `db_config.yaml`：填入 MySQL `host/port/user/password/database`
2. `model_config.yaml`：填入 `base_url`, `api_key`, `llm_model_name`
3. `basic_config.yaml`：填入 Kali `hostname/port/username/password`
4. 若不使用 RAG：`enable_rag: false`
5. 若使用 RAG：`enable_rag: true` 並補齊 `kb_config.yaml` 的 Milvus 與 `kb_name`

## D. 復現流程（命令）

1. 初始化

```bash
.venv/bin/python cli.py init
```

2. 啟動 API

```bash
.venv/bin/python cli.py start --api
```

3. 啟動 WebUI

```bash
.venv/bin/python cli.py start -w
```

4. 啟動完整 VulnBot 流程

```bash
.venv/bin/python cli.py vulnbot -m 5
```

## E. 驗證指標（Pass/Fail）

1. 初始化成功指標
- 指令 `cli.py init` 無例外中止
- 日誌包含：
  - `Creating all data directories: Success.`
  - `Initializing database: Success.`
  - `Generating default configuration file: Success.`

2. 資料庫 schema 指標
- MySQL 中存在以下資料表（至少）：
  - `conversations`
  - `messages`
  - `plans`
  - `tasks`
  - `sessions`
  - `knowledge_base`
  - `knowledge_file`
  - `file_doc`

3. API 啟動指標
- 日誌出現：
  - `Application startup complete.`
  - `Uvicorn running on http://0.0.0.0:7861`

4. WebUI 啟動指標
- 日誌出現：
  - `You can now view your Streamlit app in your browser.`
  - `URL: http://0.0.0.0:8501`

5. 端到端流程指標（VulnBot）
- Pass：角色流程可推進（Collector/Scanner/Exploiter）且可持續產生 task/plan
- Fail（常見）：`model_config.yaml` 未設定導致 LLM 呼叫錯誤（如 connection error、`NoneType` 解析失敗）
